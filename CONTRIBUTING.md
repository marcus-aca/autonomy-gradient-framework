# Contributing

Thanks for contributing to the Autonomy Gradient framework.

## What You Can Submit

1. Model language improvements
- Term clarifications
- Keyword additions
- Definition precision updates

2. Blog submissions
- Field notes
- Practical implementation patterns
- Real journey retrospectives

## Blog Submission Workflow

1. Create a new Markdown file in `content/blogs/` using `content/blogs/_template.md`.
2. Open a pull request using the PR template.
3. Include concrete experience and avoid confidential or personal data.
4. Expect editorial review and possible revision requests.

## Content Requirements

- Must align to canonical model language in `model/terms.yml`
- Must be original and lawful to publish
- Must not include private, regulated, or confidential information
- Must avoid unverifiable hard metrics unless backed by public evidence

## Rights and Editorial Control

By submitting content, you represent you have rights to submit it and you grant maintainers a worldwide, non-exclusive, royalty-free, irrevocable license to use, edit, adapt, publish, distribute, and create derivative works from your submission in connection with the Autonomy Gradient framework and related products.

Contributions are collaborative. Final editorial decisions, publication decisions, canonical terminology decisions, and repository curation rights remain with the maintainers.

Maintainers may accept, reject, edit, or remove submissions at their discretion.

## Licensing Model

Default repository license: CC BY-NC-ND 4.0.

Selected implementation-oriented files include explicit additional permissions under `LICENSE_SCOPE.md`.

## Commercial, Derivative, and Exception Requests

If you need commercial rights, derivative rights, translations, or other exceptions for core framework content, open a licensing request through the repository contribution workflow. Approved exceptions must be explicit and in writing.
